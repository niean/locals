# Harness 开发流程

## 约束
本文仅供自然人使用，未经人工确认、禁止AI修改。

## 目标
- 无人值守：AI自主运行，人类参与最小化；unmanned，AFK(Away From Keyboard)

## 项目初始化

将Harness模板文件、复制到老项目中，使用以下提示词让 AI 完成实例化：
```
初始化Harness工程：提取项目现状，完成harness工程模板的实例化。如果有模糊、搞不清的事项，请先人工确认、然后再执行。
```
注意：如遇实例化遗漏，需人工指示AI、进行补全；如有固化的产品、架构、代码等规范，需人工补录到文档。

## 人机协作开发
1. 定义需求。更新到 `03-prd-specs.md`
2. 提交需求、给AI
    - 推荐方式：引用`03-prd-specs.md`需求字段、到Chat消息框。
    - 流程控制：如弱模不遵守流程，可在Chat指令后追加要求：`请严格按照 迭代步骤 执行，禁止擅自修改跳过，特别是 需求设计确认、结果验收(如果有)`
3. 确认需求。人工确认AI识别到的需求，产出物如下；如需调整、转回步骤1
    - 执行计划（AI 自主管理在 .harness/specs/、.harness/plans/，跟踪进度和技术债）
4. 实现需求。AI按照Harness规范，大部分自主执行、少量人工确认(如删除文件)。产出物如下
    - 代码（已验收通过）
    - AI知识库（主要是 .harness/knowledge/ 文档库）
5. 验收需求。查看任务报告，按需擦屁股(如补充功能测试)
6. [可选]回填知识。经过多轮迭代后，可以考虑 更新产品知识库、Harness知识库，指令和更新对象如下
    - 执行 `回填知识库`，更新对象 `.harness/knowledge/ 文档库` + `AGENTS.md`
    - 执行 `回填产品文档`，更新对象 `.harness/prd/02-prd-baseline.md` + `01-prd-sense.md`

- 建议：授予AI Chat自主执行命令的权限，消除低级别的人工确认
- 建议：不同任务，使用不同Chat窗口；差异甚大的任务，避免复用窗口。因为，上下文污染影响Coding效果
